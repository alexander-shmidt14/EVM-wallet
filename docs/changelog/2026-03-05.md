# 2026-03-05 — Fix: Incoming Transactions (Etherscan V2 Migration)

**Ветка:** `fix/incoming-transactions-diagnostics` · **PR:** #46

---

## Описание проблемы

Входящие транзакции не отображались (`0 in | 3 out`). Диагностика показала, что ETHERSCAN_API_KEY присутствует (34 символа), но Etherscan API возвращает ошибку:

```json
{"status":"0","message":"NOTOK","result":"You are using a deprecated V1 endpoint, switch to Etherscan API V2 using https://docs.etherscan.io/v2-migration"}
```

**Причина:** Etherscan отключил V1 API (deprecated с 2025-08-15).

---

## Fixed

### Etherscan V2 API Migration
- **Base URL:** `api.etherscan.io/api` → `api.etherscan.io/v2/api`
- **Параметр `chainid=1`** добавлен во все запросы (обязателен для V2)
- Затронуты: `getIncomingTransactions()` (txlist + tokentx) и `testEtherscan` IPC

### Надёжность fetch-запросов (из предыдущих коммитов)
- **`import fetch from 'cross-fetch'`** — явный named import вместо polyfill side-effect
- **`URLSearchParams`** — правильное кодирование параметров API
- **`etherscanApiKey.trim()`** — удаление `\r\n` (частая проблема на Windows)
- **`Promise.allSettled`** — одна ошибка (ETH или ERC-20) не убивает оба запроса

## Added

### Injectable Logger (`WalletLogger`)
- Интерфейс `WalletLogger { info, warn, error }` экспортируется из `wallet-core`
- Desktop передаёт `electron-log` → логи видны в `%APPDATA%/@app/desktop/logs/main.log`
- Mobile/тесты используют `console` (default)

### Диагностика в DevTools
- `wallet:getDiagnostics` — конфигурация main-процесса (ключ, RPC, whitelist, путь к логам)
- `wallet:testEtherscan` — тестовый запрос к Etherscan V2 API с полным ответом
- `loadTransactions()` автоматически вызывает оба и логирует в Console

### ERC-20 Incoming Transactions
- Полноценная поддержка входящих ERC-20 через `action=tokentx`
- Фильтрация по whitelist (`INCOMING_ERC20_WHITELIST`, default: MMA token)
- `logIndex` для дедупликации ERC-20 (один tx hash может содержать несколько transfers)
- `tokenSymbol`, `tokenDecimals`, `tokenAddress` в `TransactionInfo`

## Changed

- `WalletCore` конструктор: 5 параметров (добавлены `incomingTokenWhitelist`, `logger`)
  - Backward compatible — 4-й и 5-й параметры опциональные
- `getTransactionHistory()`: дедупликация через `txDedupKey` (hash + tokenAddress + logIndex) вместо простого hash
- `getTransactionHistory()` и `getIncomingTransactions()`: используют `this.log` вместо `console`
