## Auto-Update Feature

### Added

#### Auto-Update System
- ✨ Система автоматических обновлений через `electron-updater` + `electron-log`
- ✨ Module: `apps/desktop/src/backend/auto-updater.ts` — управление жизненным циклом обновлений
- ✨ Проверка обновлений при старте приложения (только в production)
- ✨ Периодическая проверка каждые 30 минут
- ✨ User dialogs: "Обновление доступно?" → "Готово к установке?"
- ✨ Progress tracking: taskbar progress bar + IPC event `update-progress` для renderer
- ✨ Graceful error handling: логирование ошибок без крэша приложения

#### Build & Release
- 📦 Dependencies: добавлены `electron-updater@^6.1.0`, `electron-log@^5.0.0` в `apps/desktop`
- 📦 Publish config: добавлены блоки `publish` в `apps/desktop/electron-builder-nosign.json` и `apps/desktop/package.json`
  - Provider: GitHub (публичный репозиторий, токен в конфиге не нужен)
  - Генерация `latest.yml` при сборке для auto-updater
- 🔄 GitHub Actions: обновлен workflow `.github/workflows/windows-build.yml`
  - Флаг `--publish always` при push тега `v*`
  - `GH_TOKEN` передаётся через env в workflow (`secrets.GITHUB_TOKEN`), не в конфиге
  - Отключена double-upload через softprops/action-gh-release (electron-builder публикует напрямую)

#### IPC API
- 🔌 Preload API расширена методом `onUpdateProgress(callback)` для подписки на события скачивания

#### Version
- 📌 Desktop app version обновлена: **1.0.0 → 1.1.0**

## Transaction History Feature

### Fixed (Critical)

#### wallet-core
- **🔧 Входящие транзакции**: добавлено **явное установление `direction: 'in'`** при маппинге результатов Etherscan API в `getIncomingTransactions()` — это критический фикс, без которого входящие транзакции отображались как исходящие
- **🧩 ERC-20 incoming**: добавлена загрузка входящих ERC-20 через `Etherscan action=tokentx` с whitelist фильтрацией (`INCOMING_ERC20_WHITELIST`, default = MMA)
- **🧱 Дедупликация истории**: для ERC-20 используется стабильный dedupe key (вместо только `hash`), чтобы не терять события в одном tx

#### IPC и тестирование
- **CI пайплайн**: добавлен явный тест `TransactionHistory.test.tsx` в CI стадию для гарантии что входящие транзакции работают корректно
- **Windows Build workflow**: шаг `build:main` теперь получает `ETHERSCAN_API_KEY` (+ RPC/whitelist env), чтобы скачанный CI installer имел рабочую историю входящих

### Added

#### wallet-core
- **Per-wallet хранение**: транзакции сохраняются отдельно для каждого кошелька (`transactions_v1_{address}`)
- **`getTransactionHistory(address, limit)`**: объединённая история — local (исходящие) + Etherscan (входящие), дедупликация по hash, сортировка по timestamp desc
- **`getTransactionStatus(txHash)`**: реальные данные из блокчейна — `provider.getTransactionReceipt()` + `provider.getBlockNumber()` → confirmations, status
- **`TransactionStatus`** интерфейс: `{confirmations, currentBlock, txBlock, status}`
- **`direction`** поле в `TransactionInfo`: `'in' | 'out'`
- Обновлены `sendEth()`, `erc20Transfer()` для per-wallet сохранения с `direction: 'out'`

#### IPC (main.ts + preload.ts)
- `wallet:getTransactionHistory` — основной канал с исправлением входящих транзакций
- `wallet:getTransactionStatus` — новый канал
- `wallet:getLocalTransactions` — принимает `address` параметр

#### Zustand Store
- Добавлены поля: `transactions`, `isLoadingTransactions`
- Добавлен action: `loadTransactions()`
- Автоматический сброс `transactions` при переключении/удалении кошелька

#### UI компоненты
- **`BlockConfirmationBar`** — анимированная полоса подтверждения блоков (12 блоков, Ethereum finality standard). Зелёные=подтверждено, жёлтые+пульс=текущий, серые=ожидается, красные=ошибка
- **`TransactionItem`** — строка транзакции в списке. Иконка направления (↑/↓), тип, адрес контрагента, сумма, относительное время, статус
- **`TransactionDetailPopup`** — полноэкранный попап: все детали tx, BlockConfirmationBar с real-time polling каждые 5 сек, кнопка "View on Etherscan"

#### WalletScreen
- Интеграция списка транзакций с `TransactionItem` (с исправлением входящих)
- Попап деталей при клике
- Автообновление каждые 30 сек (вместе с балансом)
- Кнопка "View All" при > 5 транзакциях

#### SendScreen
- Вызов `loadTransactions()` после успешной отправки

#### Тесты
- `TransactionHistory.test.tsx`: тесты для TransactionItem, BlockConfirmationBar, TransactionDetailPopup
- Обновлён `test/setup.ts`: моki для `getTransactionHistory`, `getTransactionStatus`

## Developer Experience

### Auto-Update
- 🎯 Auto-updater инициализируется автоматически в production
- 🎯 Development mode: updater отключен (ускорение разработки)
- 🎯 Логирование через `electron-log` с полной диагностикой
- 🎯 Progress events доступны для кастомного UI (если потребуется позже)

### Transaction History
- 🎯 Входящие транзакции видны если:
  1. ✅ `ETHERSCAN_API_KEY` установлен в `.env` или GitHub Secrets
  2. ✅ В wallet-core установлено `direction: 'in'` (фикс 2026-03-03)
  3. ✅ Компонент `TransactionItem` правильно определяет направление

## Testing

### Auto-Update
При релизе `v1.1.0`:
1. Тег `v1.1.0` pushится в GitHub
2. Windows Build запускается с `--publish always`
3. electron-builder:
   - Генерирует `latest.yml` (версия 1.1.0)
   - Загружает `.exe` и `latest.yml` в GitHub Release
4. Клиенты с версией 1.0.0:
   - Проверяют GitHub Releases
   - Обнаруживают версию 1.1.0
   - Видят диалог обновления
   - Скачивают, устанавливают, перезагружаются

## Notes

### Auto-Update
- 🔒 **Public repo**: GitHub Release доступны для скачивания без авторизации, токен в конфиге не нужен
- 🔐 **GH_TOKEN**: передаётся только через env в CI workflow при публикации релизов (не в JSON-конфигах)
- ✅ **Backward compatible**: старые версии приложения (< 1.1.0) не ломаются
- 📊 **Telemetry**: логи обновлений сохраняются в `~/.config/EVM Wallet/logs/` для диагностики

### Transaction History
- 🔑 **Etherscan API**: критически важен для входящих транзакций
- ✅ **Whitelist**: `INCOMING_ERC20_WHITELIST` фильтрует токены для производительности
- 📊 **Per-wallet storage**: изолированная история для каждого адреса
