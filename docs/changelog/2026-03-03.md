# 2026-03-03 — История транзакций

**Ветка:** `feature/transaction-history`
**Родитель:** [[changelog/_index|Changelog]]

---

## Исправления (2026-03-03 обновление)

### wallet-core
- **🔧 Входящие транзакции**: добавлено **явное установление `direction: 'in'`** при маппинге результатов Etherscan API в `getIncomingTransactions()` — это критический фикс, без которого входящие транзакции отображались как исходящие
- **🧩 ERC-20 incoming**: добавлена загрузка входящих ERC-20 через `Etherscan action=tokentx` с whitelist фильтрацией (`INCOMING_ERC20_WHITELIST`, default = MMA)
- **🧱 Дедупликация истории**: для ERC-20 используется стабильный dedupe key (вместо только `hash`), чтобы не терять события в одном tx

### IPC и тестирование
- **CI пайплайн**: добавлен явный тест `TransactionHistory.test.tsx` в CI стадию для гарантии что входящие транзакции работают корректно
- **Windows Build workflow**: шаг `build:main` теперь получает `ETHERSCAN_API_KEY` (+ RPC/whitelist env), чтобы скачанный CI installer имел рабочую историю входящих

### Документация
- **Новый файл**: [[backend/transaction-history|История транзакций]] — полное описание архитектуры входящих/исходящих транзакций, требования к `ETHERSCAN_API_KEY`, типичные ошибки и их решения
- **Обновлено**: `backend/_index.md` — добавлена ссылка на новый документ
- **Обновлено**: `guides/env-variables.md` — уточнена роль `ETHERSCAN_API_KEY` для входящих транзакций

## Новое (оригинальный список)

### wallet-core
- **Per-wallet хранение**: транзакции сохраняются отдельно для каждого кошелька (`transactions_v1_{address}`)
- **`getTransactionHistory(address, limit)`**: объединённая история — local (исходящие) + Etherscan (входящие), дедупликация по hash, сортировка по timestamp desc
- **`getTransactionStatus(txHash)`**: реальные данные из блокчейна — `provider.getTransactionReceipt()` + `provider.getBlockNumber()` → confirmations, status
- **`TransactionStatus`** интерфейс: `{confirmations, currentBlock, txBlock, status}`
- **`direction`** поле в `TransactionInfo`: `'in' | 'out'`
- Обновлены `sendEth()`, `erc20Transfer()` для per-wallet сохранения с `direction: 'out'`

### IPC (main.ts + preload.ts)
- `wallet:getTransactionHistory` — основной канал с исправлением входящих транзакций
- `wallet:getTransactionStatus` — новый канал
- `wallet:getLocalTransactions` — принимает `address` параметр

### Zustand Store
- Добавлены поля: `transactions`, `isLoadingTransactions`
- Добавлен action: `loadTransactions()`
- Автоматический сброс `transactions` при переключении/удалении кошелька

### UI компоненты
- **`BlockConfirmationBar`** — анимированная полоса подтверждения блоков (12 блоков, Ethereum finality standard). Зелёные=подтверждено, жёлтые+пульс=текущий, серые=ожидается, красные=ошибка
- **`TransactionItem`** — строка транзакции в списке. Иконка направления (↑/↓), тип, адрес контрагента, сумма, относительное время, статус
- **`TransactionDetailPopup`** — полноэкранный попап: все детали tx, BlockConfirmationBar с real-time polling каждые 5 сек, кнопка "View on Etherscan"

### WalletScreen
- Интеграция списка транзакций с `TransactionItem` (с исправлением входящих)
- Попап деталей при клике
- Автообновление каждые 30 сек (вместе с балансом)
- Кнопка "View All" при > 5 транзакциях

### SendScreen
- Вызов `loadTransactions()` после успешной отправки

### Тесты
- `TransactionHistory.test.tsx`: тесты для TransactionItem, BlockConfirmationBar, TransactionDetailPopup
- Обновлён `test/setup.ts`: моки для `getTransactionHistory`, `getTransactionStatus`

### Документация

- Обновлён `backend/ipc-reference.md` — новые каналы + типы
